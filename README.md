# Цветы в интерьере — визуализатор для B2B (MOSS)

Внутренний инструмент: загружаешь фото интерьера клиента и фото цветочной
композиции, ставишь метку на интерьере, пишешь пожелания — и получаешь
фотореалистичную картинку «как композиция будет смотреться в этом помещении».

Генерация работает на модели **Nano Banana 2** (Gemini 3.1 Flash Image) через
**OpenRouter**. Ключ хранится только на сервере и в браузер не попадает, поэтому
инструментом можно безопасно пользоваться с телефонов нескольких сотрудников.

## Что внутри
```
floral-viz/
├─ server.js          бэкенд-прокси (Express): держит ключ, ходит в OpenRouter
├─ public/index.html  фронтенд (адаптивный, под телефон) + индикатор остатка
├─ package.json
└─ .env.example       пример переменных окружения
```

## 1. Ключ и баланс OpenRouter
1. Регистрация: https://openrouter.ai
2. Пополнить баланс: Settings → Credits (минимум $5). Для России — крипта (USDC)
   или AliPay; карты, выпущенные в РФ, в Stripe не пройдут.
3. Создать ключ: https://openrouter.ai/settings/keys → Create Key. **Рекомендуется
   сразу задать на ключе кредитный лимит** — тогда в интерфейсе будет видно остаток
   и «≈ сколько генераций» осталось.
4. Стоимость одной генерации ≈ $0.07 (несколько центов). За лимит уйти нельзя —
   баланс предоплатный: когда кредиты кончатся, придёт ошибка 402 «лимит».

## 2. Запуск локально (для проверки)
Нужен Node.js 18+.
```bash
cd floral-viz
npm install
OPENROUTER_API_KEY="sk-or-ваш_ключ" npm start
```
Открыть http://localhost:3000 · проверка ключа: http://localhost:3000/api/health
· остаток: http://localhost:3000/api/balance

## 3. Деплой как внутренний сайт (для телефонов)
Любой хостинг Node, проще всего Render или Railway:
1. Залить папку `floral-viz` в репозиторий на GitHub.
2. Создать **Web Service** → выбрать репозиторий.
3. Build: `npm install` · Start: `npm start`
4. В Environment добавить `OPENROUTER_API_KEY` (и при желании `ACCESS_PASSWORD`,
   `APP_URL` со ссылкой приложения, `EST_COST_PER_IMAGE`).
5. Полученную ссылку открывают сотрудники с телефонов (можно добавить на главный экран).

## 4. Необязательно: пароль для сотрудников
Задать `ACCESS_PASSWORD` в окружении и добавить заголовок в `fetch` в `public/index.html`:
```js
headers:{ "Content-Type":"application/json", "x-access-password":"ВАШ_ПАРОЛЬ" },
```

## Что показывает интерфейс
- **Остаток баланса** и примерное число оставшихся генераций (если на ключе задан
  лимит). При низком остатке индикатор желтеет, при нуле — краснеет.
- Понятные ошибки: нет кредитов/лимит (402), проблема с ключом (401/403), слишком
  много запросов (429), сервис недоступен (5xx), нет связи с сервером, и случай,
  когда модель не вернула картинку (фильтр) — с подсказкой.

## Замечания
- Это генеративная вставка: модель пересобирает композицию под перспективу и свет
  интерьера, поэтому возможны небольшие отличия от исходного фото букета.
- На всех изображениях есть невидимая метка SynthID.
- Остаток «≈ N генераций» — приблизительный, считается от `EST_COST_PER_IMAGE`.
